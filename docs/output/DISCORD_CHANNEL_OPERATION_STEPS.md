# Discordチャンネル運用 実装手順

## 結論

チャンネル運用は今後やっていく想定です。

現在のBotは、まず手動コマンドを動かすMVPです。次の実装で、用途ごとにDiscordチャンネルを分け、LINE相談通知・今日の対応・通話ブリーフィング・リスク通知などを決まったチャンネルへ出す形にします。

## 1. Discordで作るチャンネル

Discordサーバーに以下のテキストチャンネルを作ります。

```text
00_秘書-inbox
01_今日の対応
02_LINE相談通知
03_通話ブリーフィング
04_リスク検知
05_ナレッジ蓄積
06_商品改善メモ
07_エラー通知
```

まず最低限作るなら、以下の4つからで大丈夫です。

```text
01_今日の対応
02_LINE相談通知
03_通話ブリーフィング
07_エラー通知
```

## 2. 各チャンネルの役割

`00_秘書-inbox`:
倉本さんが自由にメモ、指示、思いつきを投げる場所。

`01_今日の対応`:
毎朝の未返信LINE、通話予定、送信予定、リスク顧客を確認する場所。

`02_LINE相談通知`:
公式LINEから届いた相談を通知する親チャンネル。相談本文、返信案、NGチェック、リスク判定は顧客ごとのスレッドへ出す。

`03_通話ブリーフィング`:
通話前の準備、通話後要約、宿題、次回テーマをまとめる場所。

`04_リスク検知`:
返金、不満、依存、禁止相談などの注意案件を集約する場所。

`05_ナレッジ蓄積`:
良かった返信、成果文面、失敗例、顧客タイプ別対応を貯める場所。

`06_商品改善メモ`:
サービス改善、月守りサポート案、頻出悩みの整理に使う場所。

`07_エラー通知`:
Bot、LINE、Google Sheets、Renderでエラーが出た時の通知先。

## 3. チャンネルIDの取得方法

1. Discordアプリを開く
2. 左下の歯車を開く
3. 詳細設定で「開発者モード」をONにする
4. 作ったチャンネルを右クリック
5. 「チャンネルIDをコピー」を押す
6. `.env` に貼る

## 4. `.env` に追加する項目

```text
DISCORD_CHANNEL_SECRETARY_INBOX_ID=
DISCORD_CHANNEL_TODAY_ID=
DISCORD_CHANNEL_LINE_NOTICE_ID=
DISCORD_CHANNEL_CALL_BRIEFING_ID=
DISCORD_CHANNEL_RISK_ID=
DISCORD_CHANNEL_KNOWLEDGE_ID=
DISCORD_CHANNEL_PRODUCT_MEMO_ID=
DISCORD_CHANNEL_ERROR_ID=
```

最初は以下だけでも進められます。

```text
DISCORD_CHANNEL_TODAY_ID=
DISCORD_CHANNEL_LINE_NOTICE_ID=
DISCORD_CHANNEL_CALL_BRIEFING_ID=
DISCORD_CHANNEL_ERROR_ID=
```

## 5. コード側の実装手順

### Step 1: チャンネルIDを設定として読み込む

`discord-bot/src/utils/config.js` に、各チャンネルIDを追加します。

### Step 2: チャンネル取得ヘルパーを作る

`discord-bot/src/integrations/discordChannels.js` を作ります。

役割:
- チャンネルIDからDiscordチャンネルを取得する
- 顧客ごとのスレッドを検索する
- スレッドがなければ自動作成する
- チャンネルがない場合はエラー通知へ出す
- 投稿用の共通関数を用意する

### Step 3: コマンドごとの推奨チャンネルを決める

```text
/today -> 01_今日の対応
/brief -> 03_通話ブリーフィング
/reply -> 02_LINE相談通知 または 実行したチャンネル
/memo -> 00_秘書-inbox または 実行したチャンネル
/customer -> 実行したチャンネル
エラー -> 07_エラー通知
```

### Step 4: LINE Webhook実装時に自動通知する

LINEから相談が来たら、以下の流れにします。

```text
LINE相談受信
-> 顧客照合
-> 顧客別スレッドを検索または自動作成
-> 相談分類
-> 返信案生成
-> NGチェック
-> リスク判定
-> 02_LINE相談通知内の顧客別スレッドへ投稿
-> LINE相談履歴へ保存
```

この時点でも、公式LINEへは自動返信しません。

### Step 5: リスクだけ別チャンネルにも転送する

リスクスコアが高い場合は、`02_LINE相談通知` に加えて `04_リスク検知` にも投稿します。

### Step 6: エラー通知を共通化する

Bot内でエラーが起きたら、`07_エラー通知` に投稿します。

あわせてスプレッドシートの `エラーログ` にも保存します。

## 6. 実装順番

1. Discordにチャンネルを作る
2. チャンネルIDを `.env` に追加する
3. BotがチャンネルIDを読めるようにする
4. `/today` を `01_今日の対応` へ投稿できるようにする
5. エラーを `07_エラー通知` へ投稿できるようにする
6. LINE Webhookを作る
7. LINE相談を `02_LINE相談通知` 内の顧客別スレッドへ自動投稿する
8. リスク高の場合だけ `04_リスク検知` にも転送する
9. 通話系を `03_通話ブリーフィング` へ出す
10. ナレッジ・商品改善系を後続Phaseで追加する

## 7. 注意点

- 本名と相談内容が出るので、サーバー参加者は最小限にする
- `02_LINE相談通知` と `04_リスク検知` は特に閲覧権限を絞る
- Botに管理者権限は不要
- Botには `公開スレッドを作成` と `Threadsでメッセージを送る` が必要
- 公式LINEへの自動返信はまだ入れない
- 最初はテスト顧客で動作確認する
