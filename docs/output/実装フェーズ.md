# 実装フェーズの順番

このフェーズ表は、PDF2点を統合した実装順序です。現時点ではPhase 1の仕様化までを対象にし、コード実装は行いません。

## Phase 1: 仕様化

目的:
- PDF要件を構造化し、実装前の前提を固定する。

やること:
- 要件定義を作る。
- 作業マニュアルのタブ仕様を作る。
- スプレッドシート仕様を作る。
- 実装手順書を作る。
- 作業マニュアル本文の構成案を作る。

成果物:
- `requirements.md`
- `specs/manual_tabs_spec.yaml`
- `specs/spreadsheet_schema.yaml`
- `CODEX_IMPLEMENTATION_GUIDE.md`
- `docs/output/作業マニュアル本文構成案.md`
- `docs/output/実装フェーズ.md`

完了条件:
- 3つの自動化対象が漏れていない。
- 4つの手動対応対象が漏れていない。
- 注意書き、返金防止、NG表現が要件に入っている。
- シート仕様に列名、型、必須/任意、生成元、更新タイミング、参照先がある。

## Phase 2: ドキュメント生成

目的:
- 作業マニュアルと実装手順書を、実際に運用で読める形へ整える。

やること:
- 作業マニュアル本文をタブ別にMarkdown生成する。
- Google Docsへ反映する。
- MVPでは見出しと目次で1文書化する。
- Google Docsタブ作成・更新可否を確認する。
- 可能ならタブ単位更新案を作る。

成果物:
- 作業マニュアルGoogleドキュメント
- 作業マニュアルMarkdown
- 更新手順

完了条件:
- 各タブに目的、実施タイミング、担当者作業、AI補助、禁止事項、チェックリスト、テンプレ文がある。

## Phase 3: MVP自動化

目的:
- LINE送信前に、フォーム回答から事前鑑定Doc生成までを安定させる。

やること:
- スプレッドシート初期化。
- 事前フォーム回答取り込み。
- 顧客ID付与。
- OpenAI API接続。
- 事前鑑定シート生成。
- Googleドキュメント保存。

成果物:
- 事前鑑定Googleドキュメント
- 顧客一覧更新
- 事前ヒアリング更新

完了条件:
- ダミー顧客でDoc生成まで動く。
- LINE送信はまだ行わない。

## Phase 4: PDF化

目的:
- 顧客提出用の恋愛スピリチュアル風PDFを生成する。

やること:
- PDFテンプレート設計。
- DocからPDF変換。
- Drive保存。
- PDF URL保存。
- スマホ表示確認。

成果物:
- 事前鑑定PDF
- PDF URL

完了条件:
- PDFが開ける。
- 顧客一覧にPDF URLが保存される。
- NG表現チェックを通っている。

- LINEユーザーID保存。

- 送信管理作成。
- フォーム回答から3時間後の送信予約。
- 時間主導トリガー。
- push message送信。
- エラー保存。

成果物:
- LINE送信管理
- PDFリンク送信
- 送信ログ

完了条件:
- テストでは3分後送信が成功する。
- 送信済みステータスが残る。
- エラー時にログが残る。
