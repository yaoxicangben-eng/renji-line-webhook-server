# 縁結び覚醒プログラム 要件定義

作成日: 2026-06-21
対象資料:
- 恋愛コーチング_自動化マニュアル.pdf
- 恋愛コーチング_自動化マニュアル②.pdf

## 1. サービスの全体像

縁結び覚醒プログラムは、購入者限定の公式LINEを入口に、恋愛コーチング、鑑定文生成、LINE伴走、週1回通話、ワーク、最終行動設計書を組み合わせて提供するサービスである。

自動化の中心は、Googleフォームで取得した情報をGoogleスプレッドシートに集約し、OpenAI APIで文章を生成し、蓮司確認用のGoogleドキュメントと顧客提出用PDFを作成し、公式LINEでPDFリンクを送る運用である。

全体フロー:
1. 購入者が購入者限定LINEに登録する。
2. LINE登録直後に注意書きPDFリンクと事前フォームURLを送る。
3. 顧客がGoogleフォームに回答する。
4. 回答内容をGoogleスプレッドシートへ保存する。
5. OpenAI APIで事前鑑定シートを生成する。
6. 蓮司確認用Googleドキュメントを作成する。
7. 顧客提出用PDFを作成し、Google Driveへ保存する。
8. フォーム回答から3時間後に公式LINEへPDFリンクを送る。
9. 期間中のLINE相談、通話メモ、ワーク回答を顧客IDで蓄積する。
10. 最終回後に今後30日間の恋愛行動設計書を生成し、確認後にPDFリンクを送る。

## 2. 自動化対象

### 2.1 初回のみ事前鑑定シート

目的:
- 購入直後に「個別に見てもらえている」安心感を作る。
- 初回通話前に、本人、相手、関係性、不安、理想状態を整理する。

入力:
- 事前ヒアリングフォーム回答
- 顧客ID
- LINEユーザーID

出力:
- 蓮司確認用Googleドキュメント
- 顧客提出用の恋愛スピリチュアル風PDF
- 公式LINEで送るPDFリンク

自動化範囲:
- フォーム回答取り込み
- AI生成
- Googleドキュメント作成
- PDF化
- 送信予約
- 3時間後LINE送信

### 2.2 心と行動を整えるワーク

目的:
- 不安、追いLINE、感情的な連投、衝動行動を防ぐ。
- 行動前に状況、感情、望む未来を整理させる。

入力:
- ワークフォーム回答
- LINE相談内容
- 顧客ID

出力:
- AI整理結果
- NG行動
- 推奨行動
- LINE文面案
- 蓮司確認用の履歴

自動化範囲:
- ワーク回答取り込み
- AI整理結果生成
- ワーク回答履歴への保存
- 必要に応じた返信案作成

重要ルール:
- ワーク結果は原則として自動送信しない。
- 恋愛不安が強い場面ほど、AI下書き後に蓮司が確認する。

### 2.3 今後30日間の恋愛行動設計書

目的:
- 1ヶ月の伴走終了後、連絡頻度、距離感、不安対策、次の一手に迷わない状態を作る。
- 顧客満足度と継続提案への納得感を高める。

入力:
- 事前鑑定シート
- 4回分の通話メモ
- LINE相談履歴
- ワーク回答履歴
- 彼の反応の変化
- 相談者の感情変化

出力:
- 蓮司確認用Googleドキュメント
- 顧客提出用の恋愛スピリチュアル風PDF
- 公式LINEで送るPDFリンク

自動化範囲:
- 顧客IDによる素材収集
- AI生成
- Googleドキュメント作成
- PDF化
- 送信管理への記録

送信ルール:
- 最終設計書は蓮司が内容確認・修正した後に送信する。

## 3. 手動対応対象

### 3.1 週1LINE通話

対象:
- 30分 x 4回

マニュアル化する内容:
- 各回の目的
- 時間配分
- 最初に答える質問
- 7日間の行動決定
- 通話後メモ
- 次回テーマ

AI補助:
- 通話文字起こし
- 要約
- 宿題抽出
- 次回テーマ案作成

### 3.2 公式LINEチャット伴走

マニュアル化する内容:
- 相談テンプレート
- 共感、整理、見立て、行動、安心の返信型
- 対応優先度
- 対応上限
- NG対応

AI補助:
- 相談分類
- 返信下書き
- NG表現チェック
- 対応ログ化

### 3.3 彼の本音・現在地読み解き

マニュアル化する内容:
- 事実、感情、距離感、障害、次の一手の5層で見立てる。
- 断定を避ける言い回しを使う。

禁止:
- 相手の気持ちを断定する。
- 復縁、連絡、結婚、恋愛成就を保証する。

### 3.4 LINE文面・行動改善

マニュアル化する内容:
- 重さ
- 要求
- タイミング
- 目的
- 余白

AI補助:
- 添削
- 修正文案
- 送信タイミング案
- 送信後の行動案

詳細は `specs/spreadsheet_schema.yaml` に定義する。

## 7. LINE連携要件

必須要件:
- WebhookでLINEユーザーIDを取得する。
- LINEユーザーIDと顧客IDを紐づける。
- LINE登録直後に注意書きPDFリンクとフォームURLを送る。
- PDFファイルを直接添付せず、PDFリンクを送る。
- 3時間後送信はreply messageではなくpush messageを使う。
- 送信予定、送信済み、エラーを送信管理に残す。
- 重要返信はAI下書き後に蓮司が確認する。

注意:
- フォーム回答とLINEユーザーの紐づけミスを防ぐため、フォームURLに顧客IDを付与する。
- Driveリンク共有は個人情報漏えいリスクを考慮して設計する。

## 8. PDF生成要件

作成対象:
- 注意書きPDF
- 事前鑑定シートPDF
- 今後30日間の恋愛行動設計書PDF

保存:
- Google Driveに保存する。
- 顧客一覧と送信管理にURLを記録する。

形式:
- 顧客向けは恋愛スピリチュアル風デザイン。
- 蓮司確認用は編集可能なGoogleドキュメント。

禁止表現:
- 必ず復縁
- 絶対連絡が来る
- 確実に恋愛成就する
- 彼はあなたを愛している
- 相手を操作する助言
- 依存を強める表現

## 9. 注意書き・返金防止要件

注意書きPDFは、購入者限定LINE登録直後、フォーム入力前に必ず配布する。

入れる内容:
- サービス目的
- 結果保証なし
- 対応範囲
- 返信条件
- 対応不可領域
- 返金条件
- 日程変更
- 無断キャンセル時の扱い
- 守秘義務
- 個人情報の扱い

法務注意:
- 返金条件やキャンセル規定は、最終公開前に行政書士または弁護士等の専門家確認を推奨する。
- 販売ページ、決済ページ、購入後LINE、注意書きPDFで内容を一致させる。

## 10. 実装上のリスクと確認事項

### 10.1 技術リスク

- Google Docsタブの自動作成・更新可否は実装前に公式仕様と実行テストで確認が必要。
- LINEにPDFを直接送る設計にしないこと。
- 3時間後送信は重複送信防止が必要。
- GASの時間主導トリガーは実行間隔と失敗時再送の設計が必要。
- Drive共有設定のミスにより個人情報が漏れる可能性がある。
- OpenAI APIエラー、LINE APIエラー、PDF生成エラーをログ化する必要がある。

### 10.2 運用リスク

- AI生成文に断定・保証・操作系表現が混ざる可能性がある。
- ワーク回答を自動送信すると、不安が強い顧客に不適切な返信を送るリスクがある。
- 顧客IDとLINEユーザーIDの紐づけミスにより誤送信が起きる可能性がある。
- 返金条件が媒体ごとに違うとトラブルになる。

### 10.3 未確定事項

- 公式LINEアカウントの作成状況
- LINE Messaging APIのチャネルアクセストークン
- Webhook公開URL
- OpenAI APIキー
- Googleフォームの最終質問項目
- 顧客IDの採番ルール
- PDFデザインテンプレート
- Drive保存先フォルダ
- PDFリンクの共有範囲
- エラー通知先メールアドレス
- テスト用LINEユーザー
- 注意書きPDFの最終文面
- 法務確認の担当者
